1 — Start the server

SSH into the robot, then activate the environment and launch the recording server:

# Kill any existing server first
pkill -f record_server.py

# Activate environment
source /home/user/robot/.venv-lerobot/bin/activate
cd /home/user/robot

# Start (single arm, execute mode)
python record_server.py \
  --arm0-port /dev/ttyACM0 \
  --arm0-channel can0 \
  --execute

# Start (dual arm, execute mode)
python record_server.py \
  --arm0-port /dev/ttyACM0 \
  --arm0-channel can0 \
  --arm1-port /dev/ttyACM1 \
  --arm1-channel can1 \
  --execute

Then open http://<robot-ip>:5000 in your browser. Find the robot IP with:

hostname -I | awk '{print $1}'
Always use http://, not https://. If the page looks stale, hard-refresh with Cmd+Shift+R (Mac) or Ctrl+Shift+R.

2 — Engage the leader arm

  1. On the Control tab, confirm the Leader 0 dot in the header is green — this means the SO-101 serial connection is live.
  2. Position the SO-101 leader arm in a comfortable starting pose.
  3. Click Engage Leader on the YAM 0 panel.
  4. The YAM panel border turns green and the badge changes to ACTIVE — the YAM is now following the leader.
Engage only when the leader arm is already in the pose you want the YAM to start from — engaging snaps the YAM to the current leader position.

3 — Record a demonstration

  1. Type a short task description in the Task description field (e.g. pick up red block and place in bowl).
  2. Click ⏺ Start Recording. A red pulsing banner appears at the top — recording is live.
  3. Perform the task with the leader arm. The frame counter in the banner updates in real time.
  4. Click ⏹ Stop Recording when the task is complete.
  5. A green toast notification confirms the save: Episode #N saved — X frames.
If you make a mistake mid-episode, click ✕ Discard to throw away the current take and start fresh — no episode is written.

4 — View recordings

  1. Click the Recordings tab.
  2. Select an episode from the left sidebar. Click ↻ Refresh if a recent episode is missing.
  3. Use the Wrist 0 / Wrist 1 / Scene tabs to switch camera angles.
  4. Scrub the video timeline and click Add Note to annotate a moment — notes are saved alongside the dataset.

5 — Disengage & shut down

  1. Click Disengage to put the YAM back into gravity-compensation mode before moving the arm by hand.
  2. Stop the server with Ctrl+C in the terminal — any recording in progress is auto-saved on shutdown.

Troubleshooting

Leader 0 dot is redWrong serial port or SO-101 not powered. Check ls /dev/ttyACM* and update --arm0-port.
Engage button disabled (strikethrough cursor)Leader dot is red — arm not connected. Fix the port first.
Start Recording → HTTP 500Dataset failed to init. Check server terminal for [ERROR] lines. Usually fixed by deleting ./recordings/ and restarting.
Episode not showing in RecordingsClick ↻ Refresh. If still missing, check server logs for save_episode error.
Video won't playBrowser must support AV1. Use Chrome 90+ or Firefox 90+.
